## 1. Bekannter Bug (vor Refactor dokumentieren)

`mainWindow` ist in `main.mjs` nirgends deklariert. `createWindow()` (Zeile 81) speichert das BrowserWindow nicht. `registerPipelineHandlers(ipcMain, () => mainWindow)` (Zeile 1361) gibt daher immer `undefined` zurueck. `sync:run-full-pipeline` kann deshalb keine Progress-Events senden (`mainWindow?.webContents?.send(...)` wird nie aufgerufen). Muss vor oder waehrend M5 gefixt werden — einfach `let mainWindow; ... mainWindow = createWindow()`.

## 4. Migrations-Phasen-Vorschlag

### Phase A — Core-Infrastruktur (low-risk, keine Handler-Aenderungen)
Migrieren: `toErrorMessage`, `resolveBundledPath`, `deriveAppBundlePath`, `computeBuildId`, `buildRuntimeConfig`, `resolveConfig`/`DEFAULTS` nach `@bpdjs/core/config` + `@bpdjs/core/log`. main.mjs importiert weiterhin aus `@bpdjs/core`, kein Verhalten aendert sich.

### Phase B — IPC-Router (low-risk, strukturell)
`ipcHandle()`-Wrapper nach `@bpdjs/ipc-router` extrahieren. 27 bestehende `ipcHandle()`-Aufrufe auf den neuen Router umstellen — rein mechanisch, keine Logik-Aenderung.

### Phase C — Lese-Handler (low-risk, hoher Erkenntnisgewinn)
Alle `low`-Risiko-Handler in folgende Pakete migrieren: `analysis:*`, `cache:*`, `engine:discover*`, `engine:inspect-schema`, `labels:*`, `scanner:list-*`, `scanner:get-*`, `unified:classify-cache`, `unified:export-m3u`, `unified:engine-diff`, `engine-analyze:*` (ausser load-playlist-tracks), `export:generate`, `export:save-playlist-local`, `sync:get-*`, `sync:check-lexicon`, `sync:check-djplaylists`.

### Phase D — Settings und File-Manager (medium-risk)
`sync:save-auth`, `sync:save-soundiiz-auth`, `sync:save-presets`, `sync:get-presets`, `sync:load-scoring-data`, `resolveLabelsDbPath` nach `@bpdjs/settings` + `@bpdjs/file-manager`. Setzt Phase A voraus (Pfad-Resolver).

### Phase E — High-Risk (am Ende, einzeln testen)
- `scoring:merge-engine-apply` + `scoring:merge-engine-preview` (Python, Backup, File-I/O)
- `engine:import-streaming` (Engine-DB-Schreiben via Python)
- `playlist:*` + `recommendations:for-track` (Beatport-Auth, XhrClient)
- `sync:djpl-*` + `sync:scrape-djplaylists` + `sync:djplaylists-to-lexicon-all` (Browser-Automation)
- `auth:*` — bleibt dauerhaft in main.mjs, wird nicht migriert
