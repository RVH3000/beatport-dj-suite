# Changelog

Alle nennenswerten Änderungen am Projekt werden in dieser Datei dokumentiert.

Das Projekt folgt [Semantic Versioning](https://semver.org/lang/de/) (MAJOR.MINOR.PATCH) und orientiert sich am Format von [Keep a Changelog](https://keepachangelog.com/de/1.0.0/).

Für alle Änderungen **vor v4.1.0** siehe [CHANGELOG-HISTORY.md](./CHANGELOG-HISTORY.md).

Ab v4.1.0 wird dieser Changelog automatisch von [`commit-and-tag-version`](https://github.com/absolute-version/commit-and-tag-version) auf Basis der Conventional-Commit-Nachrichten gepflegt. Siehe [`docs/RELEASE-WORKFLOW.md`](./docs/RELEASE-WORKFLOW.md) für Details zum Ablauf.
